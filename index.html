關於我

目前主要投入於前端開發與網頁體驗設計，包含畫面切版、RWD 響應式設計，以及將設計稿轉換為穩定可維護的前端程式碼。同時也關注教學與知識整理，希望把自己熟悉的工具與實務經驗，轉換成清楚易懂的說明，協助團隊或學習者更快上手。

過去的背景讓我習慣在溝通中釐清需求、拆解問題，並一步步說明思考過程。我偏好把抽象的想法具體化，無論是規格文件、Wireframe 還是程式架構圖，都希望用「講得清楚、看得懂」的方式，讓一起合作的人可以放心把事情交給我。

  • 協助規劃與實作乾淨、可維護的前端結構與樣式（HTML / CSS / RWD）。
  • 將複雜需求拆解成清楚的任務步驟，並以文件或教學方式說明實作細節。
  • 在設計、產品與工程之間協調，找到兼顧體驗與可實作性的折衷方案。

技能（Skills 區占位，之後會用 Section SDD 替換）

這裡目前是 Skills 區的占位內容，未來將用來清楚整理前端相關技能（例如：HTML、CSS、JavaScript、前端框架、工具鏈）、後端或 DevOps 能力，以及其他軟實力。實際的技能列表與熟練度呈現方式，將交由 Section SDD 的 Prompt 生成。

預計放「前端技術」群組

例如：HTML / CSS / RWD / JavaScript / TypeScript / 常用前端框架與 Build 工具……等，以條列或標籤形式呈現，方便閱覽。

預計放「相關後端與工具」群組

例如：基礎後端框架、API 串接經驗、資料庫、Git / GitHub、CI/CD、測試工具等，可根據實際背景調整分類名稱與內容。

預計放「軟實力與其他」群組

例如：團隊合作、溝通能力、文件撰寫、跨部門協作、語言能力，以及其他與前端工作高度相關的優勢。

之後會用 Section SDD 的 Prompt 產生新的 Skills section。請開發者把新的 section 程式碼「整段」貼在 SECTION:SKILLS-START / SECTION:SKILLS-END 這兩個註解中間，完全取代現在這一區，而不是只貼入部分 HTML。
